# Human Plan Review

Ревью Human Plan до реализации. Цель — сделать спецификацию достаточно ясной для Agent Plan и кода.

Не реализуй код и не редактируй план без явной просьбы пользователя.

## Когда использовать

- Пользователь просит проверить `docs/<feature>.md`, Human Plan, spec или план до реализации.
- Пользователь просит найти вопросы, непонятные места, потенциальные несостыковки.
- Пользователь просит проверить релевантный код, чтобы понять, реалистичен ли план.
- Перед переходом от Human Plan к Agent Plan.

## Источники фактов

1. Указанный Human Plan или spec-файл.
2. Локальные инструкции проекта: `AGENTS.md`, `CLAUDE.md`, методология docs.
3. Релевантный код, конфиги, схемы, тесты и текущая документация.
4. Слова пользователя как источник намерения.

## Workflow

1. Прочитай указанный план. Если файл не указан и его нельзя найти уверенно, спроси один конкретный вопрос.
2. Вытащи из плана целевое поведение, scope, out of scope и валидацию.
3. Проверь только релевантный код и docs, если без них нельзя оценить план.
4. Ищи:
   - неясный или расплывчатый scope;
   - противоречия внутри плана;
   - противоречия текущему коду или архитектуре;
   - отсутствующие edge cases и состояния ошибок;
   - слабую или ненаблюдаемую валидацию;
   - решения, которые уже залезают в Agent Plan без нужды.
5. Для каждого существенного пункта предложи 1-3 варианта решения с кратким обоснованием.
6. Если блокеров нет, скажи это явно и назови остаточные мелкие риски, если они есть.

## Приоритеты

1. Блокеры, из-за которых реализация может пойти не туда.
2. Вопросы, которые меняют поведение, scope или критерий приёмки.
3. Несостыковки с текущим кодом.
4. Улучшения формулировок и валидации.

Не придирайся к стилю текста, если смысл устойчив.

## Формат ответа

Начинай с короткого вердикта:

- `Готов к Agent Plan` — если блокеров нет.
- `Нужен ещё раунд Human Plan` — если есть вопросы, меняющие поведение или scope.

Дальше используй секции по необходимости:

- `Блокеры`
- `Вопросы`
- `Несостыковки`
- `Варианты решений`
- `Можно передавать в Agent Plan`

Для каждого пункта указывай, почему это важно и что выбрать дальше.
